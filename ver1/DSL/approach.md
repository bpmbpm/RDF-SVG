# Подход к DSL для SVG-RDF Viewer

## Цель

DSL в этом проекте остается браузерным JavaScript, но код приложения пишется через
крупные доменные операции: `DSL.tree.render()`, `DSL.diagram.open()`,
`DSL.rdf.extractElement()`, `DSL.viewport.zoom()`. Такой слой отделяет сценарий работы
VAD-редактора от деталей DOM, событий мыши, RDF/XML и состояния.

## Принципы

- Функциональные группы соответствуют предметным областям: файлы, дерево, диаграмма,
  RDF, область просмотра и UI.
- Состояние хранится централизованно в `App`, а функции DSL получают данные из state и
  возвращают новый экран через render-функции.
- RDF считается источником структуры: связь диаграмм берется из `vad:hasParentObj`,
  список объектов берется из SVG-групп `.vad-element`.
- DSL должен читаться как сценарий: загрузить файл, пересобрать treeView, открыть
  диаграмму, сфокусировать объект, показать свойства.
- Реализация не вводит сборщик и внешние зависимости, потому что текущая версия
  проекта распространяется как статический HTML.

## Похожие подходы

- jQuery-плагины и imperative UI: дают быстрый доступ к DOM, но логика быстро
  смешивается с обработчиками событий. Здесь DOM спрятан за группами DSL.
- Redux/Elm architecture: централизуют состояние и делают изменения предсказуемыми.
  В проекте использована упрощенная модель без action reducers, чтобы сохранить
  статический HTML.
- Mermaid/PlantUML/Graphviz DOT: описывают диаграмму декларативно и генерируют
  визуальное представление. RDF-SVG DSL отличается тем, что работает с уже готовым SVG
  и связывает визуальные элементы с RDF-метаданными.
- ARIS/VAD-моделирование: использует типизированные объекты и связи между ними. В этом
  проекте типы и связи представлены SVG-примитивами и RDF-предикатами.

## Направление развития

- Вынести DSL-группы из `index.html` в отдельные ES-модули.
- Добавить декларативный формат генерации VAD-схем из JSON/RDF в SVG-примитивы.
- Подключить RDF/TriG-парсер, чтобы `hasParentObj` и другие связи читались не только из
  RDF/XML, но и из TriG-блоков.
- Добавить undo/redo для операций RDF-редактора.
- Покрыть DSL-функции unit-тестами после появления сборки или test runner.
